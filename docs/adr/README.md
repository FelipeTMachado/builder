# Architecture Decision Records (ADRs)

Este diretório contém os registros de decisões arquiteturais do nosso projeto (Web 3D Builder). 

## O que é um ADR?
Um Architecture Decision Record é um documento curto que captura uma decisão arquitetural importante, junto com o seu contexto e as consequências dessa decisão.

Usaremos o formato padrão:
- **Título**: O que estamos decidindo.
- **Status**: Proposto / Aceito / Rejeitado / Descontinuado.
- **Contexto**: Qual é a força motriz, problema ou cenário que nos força a decidir.
- **Decisão**: A mudança que estamos fazendo.
- **Consequências**: O que melhora ou piora após essa decisão (trade-offs).
